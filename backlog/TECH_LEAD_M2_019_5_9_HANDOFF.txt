## ARQUIVOS ALTERADOS

### [NEW] scripts/model2/entry_rl_filter.py (391 linhas)
Propósito: Estágio intermediário que filtra CREATED signals via predição RL
Status: CRIADO, totalmente funcional
Função pública: run_entry_rl_filter(model2_db_path, symbol, timeframe, limit,
                                     dry_run, output_dir, threshold=0.55, manager_cls=None)
Output: JSON com {fallback, pass_through, cancelled_neutral, cancelled_contradiction,
                   enriched_match, erro}

Decisões de Design:
- Threshold padrão 0.55 (configurável por parameter)
- Confidence < threshold → CREATED preservado (fallback conservador)
- action=NEUTRAL → CANCELLED imediato
- action contradiz signal_side → CANCELLED imediato
- action == signal_side → CREATED enriched (payload auditora)
- Model ausente ou erro → CREATED preservado (fail-safe)

Auditoria em Payload:
6 campos imutáveis em nested `rl_entry_filter` dict:
  - decision (int: 0|1|2)
  - confidence (float: 0.0-1.0)
  - threshold (float: 0.55)
  - reason (str: rl_entry_lowconf|rl_entry_neutral|rl_entry_match|rl_entry_contradiction)
  - agent_version (str: SubAgentManager class name)
  - decision_timestamp (int: ms since epoch)

Transações:
- Atomic write + UPDATE com locked transaction
- Rollback em dry_run
- Preserva idempotência por decision_id
