Execute o fluxo completo do agente:
1. Reproducao independente — executar localmente:
   - `pytest -q tests/test_<modulo>.py`
   - `mypy --strict <modulo>.py`
   - `pytest -q tests/` (regressoes)
2. Revisao de codigo: tipos, nomenclatura, logging, DRY, seguranca, guardrails
3. Revisao de testes: cobertura 100% funcoes publicas, AAA, determinismo
4. Verificar conformidade com `docs/REGRAS_DE_NEGOCIO.md`
5. Decisao BINARIA: APROVADO ou DEVOLVIDO_PARA_REVISAO
6. Atualizar `docs/BACKLOG.md` + comentario `TL: <resumo ate 150 chars>`
7. Gerar APENAS o prompt para o proximo agente:
   - APROVADO → prompt para `7.doc-advocate` (schema compacto, ate 1800 chars)
   - DEVOLVIDO → prompt para `5.software-engineer` com itens especificos

Schema handoff TL->DA (obrigatorio quando APROVADO):
- id, decisao, status_backlog, resumo_tecnico (ate 300 chars)
- docs_impactadas (1-8), evidencias (ate 8 linhas)
- guardrails: risk_gate, circuit_breaker, decision_id
- pendencias, Gate_payload (tamanho_chars, limite_chars: 1800)

Guardrails:
- Qualquer guardrail ausente → DEVOLVIDO automatico
- Nunca aprovar sem executar testes localmente
- Decisao parcial nao existe: APROVADO ou DEVOLVIDO
