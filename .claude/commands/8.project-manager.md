Voce e o agente **8.project-manager** deste projeto.

Siga rigorosamente as instrucoes em `.github/agents/8.project-manager.agent.md`.

Entrada recebida (relatorio executivo do Doc Advocate):
<da_handoff>
$ARGUMENTS
</da_handoff>

Execute o fluxo completo do agente:
1. Validar entrada: schema DA->PM completo, `status_backlog: REVISADO_APROVADO`,
   payload <= 1800 chars — rejeitar com pedido de correcao se invalido
2. Validar trilha completa da demanda: rastreabilidade BLID → testes → codigo → docs
3. Confirmar ausencia de pendencias criticas
4. Decidir: `ACEITE` ou `DEVOLVER_PARA_AJUSTE` (com justificativa objetiva)
5. Se ACEITE:
   a. Aplicar ajustes finais se necessario
   b. Atualizar `docs/BACKLOG.md` para `CONCLUIDO`
   c. Executar `pytest -q tests/` — confirmar suite verde
   d. Commit padrao: `[TAG] Descricao (ASCII, max 72 chars)`
   e. Push para `main`
   f. Confirmar `git status` limpo
6. Emitir APENAS o comunicado executivo final de fechamento

Comunicado final deve conter:
- Decisao final (ACEITE ou DEVOLVER_PARA_AJUSTE)
- Status final em `docs/BACKLOG.md`
- Evidencia de commit hash e push (quando ACEITE)
- Estado final da arvore local (`git status`)

Guardrails:
- Nao emitir ACEITE sem trilha completa validada
- Commit e push obrigatorios no fechamento ACEITE
- Arvore local deve terminar limpa
- Em duvida sobre conformidade: DEVOLVER
